# [📈 Live Status](https://coreeng.github.io/cecg-status): <!--live status--> **🟩 All systems operational**

Public status page for the CECG core platform applications. Live at **[status.cecg.io](https://status.cecg.io)** (during Phase 1 rollout, also at [coreeng.github.io/cecg-status](https://coreeng.github.io/cecg-status/)).

Built on [Upptime](https://upptime.js.org/) — a GitHub Actions-powered uptime monitor with no server to run. Scheduled workflows probe each site's `/livez` endpoint every 5 minutes, commit the result back to this repo, and rebuild the static status page on GitHub Pages.

Tracks [PT-397](https://cecg-force.atlassian.net/browse/PT-397) (ISO 27001 / SOC 2 evidence for the public status page requirement).

## What's monitored

The four core platform applications, each probed via `GET /livez`:

- `cecg.io`
- `coreplatform.io`
- `community.cecg.io`
- `osterley.cecg.io`

Live numbers are in the auto-generated table below.

## How to respond to an incident

See [`docs/incident-runbook.md`](./docs/incident-runbook.md) for detection, acknowledgement, external communication, and post-incident review procedures. This runbook is the primary operational reference and is also the procedural evidence for audits.

## How to add a new monitored site

Edit [`.upptimerc.yml`](./.upptimerc.yml), append under `sites:`:

```yaml
  - name: Display Name
    url: https://host.example.com/livez
```

Open a PR. On merge, `Setup CI` picks it up and the site appears on the page within a few minutes. See §8 of the runbook for related operations (removing a site, pausing for maintenance).

## Design and architecture

The full design specification is at [`docs/superpowers/specs/2026-04-20-upptime-cecg-status-design.md`](./docs/superpowers/specs/2026-04-20-upptime-cecg-status-design.md). It covers:

- The two-phase rollout (Phase 1: MVP on default `github.io` URL; Phase 2: `status.cecg.io` + Slack alerting + branding)
- Why Upptime satisfies the "independent of the monitored apps" requirement
- The evidence package produced for ISO 27001 / SOC auditors

## Ownership

Owned by the CECG Core Platform team. For issues or questions, open an Issue in this repo or ping the team in Slack.

Incident alerts are delivered automatically to the core platform alerts Slack channel — see the runbook for details.
